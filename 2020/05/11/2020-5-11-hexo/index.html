（持续更新）从jekyll到hexo的迁移旅途

· 2020-5-11 · 次阅读

动机

在为jekyll美化的过程中，发现有许多插件都是基于hexo，这些插件移植到jekyll的过程十分令人喷粪。加上hexo的部署过程更符合我的使用习惯，就想着什么时候能把我的博客系统搞到hexo上。

选择主题：Butterfly

开创博客框架

打开终端，先安装hexo支持，噼里啪啦：

npm install hexo-cli -g

然后打开你想要放置hexo博客主文件夹的文件夹。

cd ./yourfolder

接下来创建一个hexo模板。

hexo init yourawseomehexoblog

然后进入这个文件夹，可以看到很多东西。先别着急，还没配置完成，继续噼里啪啦：

cd yourawesomehexoblog
npm install

此时，你的hexo blog已经初步搭建完成了。

部署主题

这里git一下你所选择的主题，将它放到/theme文件夹下，我这里选择的是butterfly主题。

然后就高兴地把md文件从jekyll搬到hexo目录下了，结果发现要改的东西还挺多的（基于butterfly）：

yaml头

封面图：header-img改为top_img

副标题：subtitle改为description

删去layout

Aplayer播放器

从yaml头彻底迁出来了，需要安装Aplayer.js和meting.js支持，除此之外需要安装hexo-tag-aplayer。

cd yourawesomehexoblog
npm install --save hexo-tag-aplayer

然后需要在根目录的/config.yml下添加如下语句：

aplayer:
    meting: true

然后在md文件需要添加aplayer的地方添加如下语句：

{% meting "yourmusicid" "yourserver" "yourmusictype" "volume:0.5" "theme:#49A6F9" %}

其中yourmusicid项目需要覆盖成你的歌曲/歌单id，yourserver项目覆盖成你所使用的音乐平台，如netease，tencent等。yourmusictype项目覆盖成你所要用的「类型」，有song曲目，playlist歌单等等。
