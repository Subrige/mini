前言

因为手痒，所以会经常写一些文章，缺少一个堆放的地方，就选择了博客。先使用的是lofter，但lofter除了文章不挂在本博客内之外，还有些太开放了。因此，听说了Github Pages后，就寻求搭建了一个Blog。

部署系统：MacOS

需要的东西

这个相信网上已经有很多教程了

1.Ruby

2.Jekyll

3.Bundler

4.Github账号（废话

Tips：如果你的电脑系统为MacOS，那么MacOS终端自带Ruby，这里就省去了安装Ruby的步骤。

仓库Settings中的Github Pages处出现：

看见这个字眼让我头大了许久。因为网上的教程里都是在这里去选择“master”。

实际上，这是在版本更迭后仓库与用户名重合出现的问题，（我是这么理解的）

根本不用管它。

下载主题版本与Jekyll不符，无法在本地服务器查看

碰到这个现象，我是直接莽fetch上github去的。

网页乱码

我遇到的情况是因为网页加载的太慢了，导致部分东西没有加载出来。可以换一个浏览器，重新加载或者把主题里post的东西一类删一些。

更新的网页迟迟没有加载出来

多半是碰到了语法错误。Github会向你的邮箱发一封邮件，里面标明了语法错误出现在哪一行。
